What is Docker?
Docker is a platform that lets you package an application along with all its dependencies (libraries, configs, runtime)
into a single unit called a container.

Why Docker is Used
Problems Docker Solves:
Different environments (dev, test, production)
Dependency conflicts
Difficult setup for new developers
Deployment issues

Consistent environment
Fast deployment
Lightweight (compared to VMs)
Easy scaling (used in cloud, Kubernetes).

Docker Image
Blueprint/template of your app
Contains code + dependencies

👉 Example:
Node.js app image, Python app image

Docker Container
Running instance of an image
Like a live app

👉 Image = class
👉 Container = object

Dockerfile:
A file with instructions to build an image.

FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["npm", "start"]

Docker Hub:
Public registry to store images
Like GitHub but for Docker images

Docker Engine
The core service that runs containers

Includes:

Docker CLI
Docker Daemon

How Docker Works (Flow)
Write Dockerfile
Build Image
Run Container

docker build -t myapp .
docker run -p 3000:3000 myapp


Important Docker Commands
Basic Commands
docker --version
docker pull nginx
docker images
docker ps
docker ps -a

Run & Stop Containers
docker run nginx
docker stop <container_id>
docker rm <container_id>

Build & Run Custom App
docker build -t myapp .
docker run -d -p 5000:5000 myapp

Logs & Debugging
docker logs <container_id>
docker exec -it <container_id> bash

Docker Architecture
Components:
Docker Client → CLI (you use commands)
Docker Daemon → Runs containers
Docker Images
